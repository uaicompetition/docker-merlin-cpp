1 # Description
2 
3 Merlin is a standalong solver written in `C++` that implements state-of-the-art exact
4 and approximate algorithms for probabilistic inference over graphical models
5 including both directed and undirected models (e.g., Bayesian networks,
6 Markov Random Fields). It can be used for many applications and research in
7 bioinformatics, computer vision, or speech and language processing to name a
8 few. Merlin supports the most common probabilistic inference tasks such as
9 computing the partition function or probability of evidence (PR), posterior
10 marginals (MAP), as well as MAP (also known as maximum aposteriori or most
11 probable explanation) and marginal MAP configurations.
12 
13 Merlin implements the classic Loopy Belief Propagation (LBP) algorithm as well
14 as more advanced generalized belief propagation algorithms such as Iterative
15 Join-Graph Propagation (IJGP) and Weighted Mini-Bucket Elimination (WMB). The
16 default algorithm for computing all four probabilistic inference tasks (PR, MAR,
17 MAP, MMAP) is WMB(i). The algorithm is parameterized by an i-bound that allows
18 for a controllable tradeoff between accuracy of the results and the computational
19 cost. Larger values of the i-bound typically yield more accurate results but
20 it takes more time and memory to compute them. Selecting a large enough i-bound
21 allows for exact inference (i.e., i-bound equal to the treewidth of the model).
22 For relatively small i-bounds Merlin performs approximate inference.

25 # API
26 
27 
28 ## Merlin API
29 Class `Merlin` defined in `merlin.h` header exposes most of the functionality
30 of the solver. A graphical model is a collection of factors (or positive
31 real-valued functions) defined over subsets of variables. Variables are
32 assumed to be indexed from `0`.
33 
34 ### Methods
35 
36  bool read_model(const char* f)
37 This method loads the graphical model from a file which is specified using the
38 UAI format (see also the File Formats section). Returns `true` if successful
39 and `false` otherwise.
40 
41  bool read_evidence(const char* f)
42 This method loads the evidence variables and their corresponding observed values
43 from a file which is also specified using the UAI format. Returns `true` if
44 successful, and `false` otherwise.
45 
46  bool read_query(const char* f)
47 This method loads the query variables from a file specified using the UAI format.
48 The query variables (also known as MAX of MAP variables) are only specific to
49 Marginal MAP (MMAP) inference tasks. Returns `true` if successful, and `false`
50 otherwise.
51 
52  bool set_task(size_t t)
53 This method sets the probabilistic inference task to be solved. The possible
54 values for the `t` parameter are:
55 * `MERLIN_TASK_PR` : Partition function (probability of evidence)
56 * `MERLIN_TASK_MAR` : Posterior marginals (given evidence)
57 * `MERLIN_TASK_MAP` : Maximum aposteriori (given evidence)
58 * `MERLIN_TASK_MMAP` : Marginal MAP (given evidence)
59 
60  bool set_algorithm(size_t a)
61 This method sets the the algorithm to be used when solving the selected
62 probabilistic inference task. The possible values for the `a` parameter are:
63 * `MERLIN_ALGO_GIBBS` : Gibbs sampling
64 * `MERLIN_ALGO_LBP` : Loopy belief propagation
65 * `MERLIN_ALGO_IJGP` : Iterative join graph propagation
66 * `MERLIN_ALGO_JGLP` : Join graph linear programming
67 * `MERLIN_ALGO_WMB` : Weighted mini-bucket elimination
68 * `MERLIN_ALGO_AOBB` : AND/OR branch and bound search (not implemented yet)
69 * `MERLIN_ALGO_AOBF` : Best-first AND/OR search (not implemented yet)
70 * `MERLIN_ALGO_RBFAOO` : Recursive best-first AND/OR search (not implemented yet)
71 
72  void set_param_ibound(size_t ibound)
73 This method sets the i-bound parameter which is used by the following
74 algorithms: `WMB`, `IJGP`, `JGLP` (as well as search based ones `AOBB`, `AOBF`,
75 and `RBFAOO`). The default value is `4`.
76 
77  void set_param_iterations(size_t iter)
78 This method sets the number of iterations to be executed by the inference
79 algorithm. The parameter is specific to the following algorithms: `WMB`, `IJGP`,
80 `JGLP`, and `LBP`. The default value is `100`. For Gibbs sampling consider
81 runnig several thousands of iterations.
82 
83  void set_param_samples(size_t s)
84 This method sets the number of samples to be generated in each iteration of the
85 `GIBBS` sampling algorithm. The default value is `100`.
86 
87  void run()
88 This method runs the inference algorithm for the selected task on the input
89 graphical model and evidence (if any). The output is generated into a file
90 specified using the UAI format. The name of the output file is obtained from
91 the input file augmented with the `task.out` suffix, where `task` corresponds
92 to one of the follwing: `PR`, `MAR`, `MAP`, or `MMAP`.

120 # File Formats
121 
122 ## Input File Format
123 
124 Merlin uses a simple text file format which is specified below to describe a
125 problem instances (i.e., graphical model). The format is identical to the one
126 used during the UAI Inference competitions.
127 
128 ### Structure
129 The input file format consists of the following two parts, in that order:
130 
131  <Preamble>
132  <Factors>
133 
134 The contents of each section (denoted <…> above) are described in the following:
135 
136 #### Preamble
137 The description of the format will follow a simple Markov network with three
138 variables and two functions. A sample preamble for such a network is:
139 
140  MARKOV
141  3
142  2 2 3
143  2
144  2 0 1
145  2 1 2
146 
147 The preamble starts with one line denoting the type of network. Generally, this
148 can be either BAYES (if the network is a Bayesian network) or MARKOV (in case of
149 a Markov network).
150 
151 The second line contains the number of variables.
152 
153 The third line specifies the cardinalities of each variable, one at a time,
154 separated by a whitespace (note that this implies an order on the variables
155 which will be used throughout the file).
156 
157 The fourth line contains only one integer, denoting the number of cliques in the
158 problem. Then, one clique per line, the scope of each clique is given as follows:
159 The first integer in each line specifies the number of variables in the clique,
160 followed by the actual indexes of the variables. The order of this list is not
161 restricted (for Bayesian networks we assume that the child variable of the clique
162 is the last one). Note that the ordering of variables within a factor will
163 follow the order provided here.
164 
165 Referring to the example above, the first line denotes the Markov network, the
166 second line tells us the problem consists of three variables, let's refer to
167 them as `X`, `Y`, and `Z`. Their cardinalities are `2`, `2`, and `3`
168 respectively (from the third line). Line four specifies that there are 2 cliques.
169 The first clique is `X,Y`, while the second clique is `Y,Z`. Note that
170 variables are indexed starting with `0`.
171 
172 #### Factors
173 Each factor is specified by giving its full table (i.e, specifying a
174 non-negative real value for each assignment). The order of the factors is
175 identical to the one in which they were introduced in the preamble, the first
176 variable has the role of the 'most significant' digit. For each factor table,
177 first the number of entries is given (this should be equal to the product of the
178 domain sizes of the variables in the scope). Then, one by one, separated by
179 whitespace, the values for each assignment to the variables in the factor's
180 scope are enumerated. Tuples are implicitly assumed in ascending order, with
181 the last variable in the scope as the `least significant`. To illustrate, we
182 continue with our Markov network example from above, let's assume the following
183 conditional probability tables:
184 
185  X | P(X)
186  0 | 0.436
187  1 | 0.564
188 
189  X Y | P(Y,X)
190  0 0 | 0.128
191  0 1 | 0.872
192  1 0 | 0.920
193  1 1 | 0.080
194 
195  Y Z | P(Z,Y)
196  0 0 | 0.210
197  0 1 | 0.333
198  0 2 | 0.457
199  1 0 | 0.811
200  1 1 | 0.000
201  1 2 | 0.189
202 
203 Then we have the corresponding file content:
204 
205  2
206  0.436 0.564
207 
208  4
209  0.128 0.872
210  0.920 0.080
211 
212  6
213  0.210 0.333 0.457
214  0.811 0.000 0.189
215 
216 Note that line breaks and empty lines are effectively just a whitespace,
217 exactly like plain spaces “ ”. They are used here to improve readability.
218

220 ## Evidence File Format
221 Evidence is specified in a separate file. The evidence file consists of a single
222 line. The line will begin with the number of observed variables in the sample,
223 followed by pairs of variable and its observed value. The indexes correspond to
224 the ones implied by the original problem file.
225 
226 If, for our example Markov network, `Y` has been observed as having its first
227 value and `Z` with its second value, the evidence file would contain the
228 following line:
229 
230  2 1 0 2 1
231 
232 ## Query File Format
233 Query variables for Marginal MAP inference are specified in a separate file.
234 The query file consists of a single line. The line will begin with the number of
235 query variables, followed by the indexes of the query variables. The indexes
236 correspond to the ones implied by the original problem file.
237 
238 If, for our example Markov network, we want to use `Y` as the query variable
239 the query file would contain the following line:
240 
241  1 1
242 
243 ## Output File Format
244 The first line must contain only the task solved: `PR|MAP|MAR|MMAP`. The rest
245 of the file will contain the solution for the task. Solvers can write more then
246 one solution by writing `-BEGIN-` at the head of the new solution. In the
247 example below the task we choose is `PR`. We have two solutions. The format of
248 the `<SOLUTION>` part will be described below.
249 
250  PR
251  <SOLUTION>
252  -BEGIN-
253  <SOLUTION>
254 
255 The solution format are as follows depending on the task:
256 
257 ### Partition function `PR`
258 Line with the value of the log10 of the partition function. For example, an
259 approximation `log10 Pr(e) = -0.2008`which is known to be an upper bound may
260 have a solution line:
261 
262  -0.2008
263 ### Maximum aposteriori `MAP`
264 A space separated line that includes:
265 * the number `n` of model variables, and
266 * the MAP instantiation, a list of value indices for all `n` variables.
267 
268 For example, an input model with 3 binary variables may have a solution line:
269 
270  3 0 1 0
271 
272 ### Marginals `MAR`
273 A space separated line that includes:
274 * the number of variables `n` in the model, and
275 * a list of marginal approximations of all the variables. For each variable
276 its cardinality is first stated, then the probability of each state is stated.
277 The order of the variables is the same as in the model, all data is space
278 separated.
279 
280 For example, a model with `3` variables, with cardinalities of `2`, `2`, `3`
281 respectively. The solution might look like this:
282 
283  3 2 0.1 0.9 2 0.3 0.7 3 0.2 0.2 0.6
284 
285 ### Marginal MAP `MMAP`
286 A space separated line that includes:
287 * the number `q` of query (or MAP) variables, and
288 * their most probable instantiation, a list of variable value pairs for all `q`
289 variables.
290 
291 For example, if the solution is an assignment of `0`, `1` and `0` to three query
292 variables indexed by `2` `3` and `4` respectively, the solution will look as follows:
293 
294  3 2 0 3 1 4 0
